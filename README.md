# ระบบสรุปวัสดุ BOQ — Smart Summary Pro

เว็บแอป (Next.js) สำหรับแยก/รวมวัสดุจากไฟล์ BOQ — เวอร์ชันเว็บของโปรแกรม
เดสก์ท็อปเดิม (`boq_report.py` + `รัน-สรุปวัสดุ.bat`)

## โปรแกรมนี้ทำอะไร

อัปโหลดไฟล์ BOQ (`.xlsx`) แล้วระบบจะ:

- แกะรายการวัสดุ/งานจากทุกชีตงานย่อย (มองหาหัวตาราง `ลำดับ / รายการ / หน่วย`)
- **รวมของซ้ำ** ที่ใช้ข้ามหลายอาคารให้เป็นรายการเดียว พร้อมนับว่าใช้กี่อาคาร
- **แยกตามหมวดงาน** (โครงสร้าง / สถาปัตย์ / หลังคา / ไฟฟ้า / โยธา ฯลฯ) ตามรหัสหมวด `X.Y.Z`
- แสดงแดชบอร์ด: มูลค่ารวม, จำนวนรายการ, จำนวนชีต, กราฟสัดส่วนต้นทุน, ตารางต่อหมวด
- มีบทสรุปผู้บริหารด้วย AI (Genkit)
- ปุ่มดาวน์โหลด **สร้างไฟล์รายงาน `[สรุปวัสดุ] ...xlsx` จริง** เหมือนโปรแกรมเดิม
  (ชีต `สรุปต่อหมวด` + ชีตแต่ละหมวด + `วัสดุรวมทุกหมวด`)

## โครงสร้างหลัก

| ไฟล์ | หน้าที่ |
| --- | --- |
| `src/lib/boq-processor.ts` | พอร์ตตรรกะจาก `boq_report.py` — แกะรายการ, จัดหมวด (`categorize`), รวมของซ้ำ (`normKey`), หาหัวตาราง (`findHeader`) |
| `src/lib/boq-report.ts` | สร้างไฟล์รายงาน `.xlsx` แบบมีสไตล์ (ExcelJS) ให้ตรงกับผลลัพธ์ของ openpyxl เดิม |
| `src/app/api/process/route.ts` | รับไฟล์ → ประมวลผล → คืนสรุป JSON + รายงาน base64 |
| `src/app/page.tsx` | แดชบอร์ด/อัปโหลด/ดาวน์โหลด |

> หมายเหตุ: ถ้าไฟล์ BOQ หน้าตาต่างจากของ กปภ. จนหาหัวตารางไม่เจอ ระบบจะแจ้ง error
> ชัดเจน (ไม่สร้างข้อมูลปลอม) — หากต้องรองรับฟอร์แมตใหม่ ให้ปรับ `findHeader`/`categorize`

## รันในเครื่อง

```bash
npm install
npm run dev      # http://localhost:9002
npm run build    # production build
```

บทสรุป AI ต้องตั้งค่า key ของ Google Generative AI (ดู `src/ai/genkit.ts`)
