# ระบบสรุปวัสดุ BOQ — Smart Summary Pro

เว็บแอป (Next.js) สำหรับแยก/รวมวัสดุจากไฟล์ BOQ — เวอร์ชันเว็บของโปรแกรม
เดสก์ท็อปเดิม (`boq_report.py` + `รัน-สรุปวัสดุ.bat`)

## โปรแกรมนี้ทำอะไร

อัปโหลดไฟล์ BOQ (`.xlsx`) แล้วระบบจะ:

- แกะรายการวัสดุ/งานจากทุกชีตงานย่อย (มองหาหัวตาราง `ลำดับ / รายการ / หน่วย`)
- **รวมของซ้ำ** ที่ใช้ข้ามหลายอาคารให้เป็นรายการเดียว พร้อมนับว่าใช้กี่อาคาร
- **แยกตามหมวดงาน** (โครงสร้าง / สถาปัตย์ / หลังคา / ไฟฟ้า / โยธา ฯลฯ) ตามรหัสหมวด `X.Y.Z`
- แสดงแดชบอร์ด: มูลค่ารวม, จำนวนรายการ, จำนวนชีต, กราฟสัดส่วนต้นทุน, ตารางต่อหมวด
- มีบทสรุปผู้บริหารด้วย AI (Genkit)
- ปุ่มดาวน์โหลด **สร้างไฟล์รายงาน `[สรุปวัสดุ] ...xlsx` จริง** เหมือนโปรแกรมเดิม
  (ชีต `สรุปต่อหมวด` + ชีตแต่ละหมวด + `วัสดุรวมทุกหมวด`)

## เครื่องมือคุมต้นทุน (Phase 1)

นอกจากสรุป/ดาวน์โหลด ยังมีเครื่องมือฝั่งเว็บสำหรับวิศวกร cost control:

- **Top Cost Drivers (Pareto 80/20)** — ชี้รายการที่กินงบสูงสุด + % สะสม
- **ตารางวัสดุแบบโต้ตอบ** — ค้นหา / กรองตามหมวด / จัดเรียงทุกคอลัมน์
- **เทียบราคาอ้างอิง → ส่วนต่าง** — กรอก "ราคากลาง/ราคาที่ตั้งไว้ต่อหน่วย" แล้วคำนวณส่วนต่าง
  (ต่อรายการ + ผลรวม) แบบเรียลไทม์ ระบบจำค่าด้วย localStorage และ **Export CSV** ได้
- ไฮไลต์รายการที่ใช้ข้ามหลายอาคาร (ช่วยตัดสินใจจัดซื้อรวม)

## โครงสร้างหลัก

| ไฟล์ | หน้าที่ |
| --- | --- |
| `src/lib/boq-processor.ts` | พอร์ตตรรกะจาก `boq_report.py` — แกะรายการ, จัดหมวด (`categorize`), รวมของซ้ำ (`normKey`), หาหัวตาราง (`findHeader`) |
| `src/lib/boq-report.ts` | สร้างไฟล์รายงาน `.xlsx` แบบมีสไตล์ (ExcelJS) ให้ตรงกับผลลัพธ์ของ openpyxl เดิม |
| `src/app/api/process/route.ts` | รับไฟล์ → ประมวลผล → คืนสรุป JSON + รายงาน base64 |
| `src/app/page.tsx` | แดชบอร์ด/อัปโหลด/ดาวน์โหลด |

### ความยืดหยุ่นในการอ่าน (robustness)

ตัวอ่านถูกปรับให้รองรับ BOQ หลายแบบมากกว่าโปรแกรมเดิม:

- **หัวตารางยืดหยุ่น** — รับคำพ้องทั้งไทย/อังกฤษ (เช่น `ที่`/`รายละเอียด`/`หน่วยนับ`/`ปริมาณ`,
  `Description`/`Unit`/`Qty`) และปรับตามตำแหน่งคอลัมน์เงินที่เลื่อน หรือคอลัมน์รวมช่องเดียว (`ค่าวัสดุรวม`)
- **เตือนแทนที่จะเดามั่ว** — ถ้าหา label คอลัมน์เงินไม่เจอ จะ *แจ้งเตือน* (ไม่เงียบ) ว่าตัวเลขอาจคลาดเคลื่อน
- **รวมของซ้ำฉลาดขึ้น** — normalize หน่วย (`ตร.ม.`=`ตารางเมตร`) และรวมชื่อเหล็กเสริมที่เขียนต่างกัน
  (`เหล็กข้ออ้อย dia 12 มม.` = `เหล็ก DB12`) โดยยัง *ไม่* รวมข้ามขนาด/เกรด/ชนิด
- ถ้าหัวตารางต่างไปมากจนหาไม่เจอจริงๆ → ขึ้น error ชัดเจน (ไม่สร้างข้อมูลปลอม)
- ถ้าไม่มีรหัสหมวด `X.Y.Z` → รวมเป็น "งานอื่นๆ" พร้อมแจ้งเตือน (ยอดรวมยังถูกต้อง)

> ปรับ/เพิ่มกฎได้ที่ `findHeader` (หัวตาราง), `categorize` (หมวดงาน), `dedupKey`/`normUnit` (รวมของซ้ำ)
